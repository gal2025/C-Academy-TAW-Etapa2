### Funcionalidades realizadas:

1. Implementação dos ficheiros HTML e CSS (de acordo com as instruções fornecidas) para o `index.html`. Layout responsivo. Organização em pastas e consistência no uso do CSS.
2. Deve ser possível fazer o registo de um novo utilizador (username/e-mail, nome, e-mail, fotografia, telemóvel, nif, morada) e login.
3. Deve ser possível visualizar o perfil/conta do utilizador logado, ou seja, a sua informação do registo.
4. Deve ser possível alternar a visualização entre o Formulário (Login/Registo) e a Área de Perfil (após Login).
5. Deve ser suportada a lógica completa de registo, login e sessão (utilizando `localStorage`). Deve ser mantido um array com a informação dos utilizadores registados.
6. Devem ser prevenidos erros de validação e respetivo tratamento dos erros (ex.: botão desativado se campos não preenchidos, mensagens de erro, etc.).
7. O admin deve poder visualizar a lista de utilizadores registados (a página HTML deverá ser criada dinamicamente utilizando o Javascript), e remover utilizador(es) seleccionado(s).
8. Uso eficiente de `localStorage`. Boa organização do código. Qualidade dos comentários.

### Extra:

9. Permitir editar e guardar a informação do perfil/conta.
